# 10 — 持久化模式：Master + Overrides

## 问题背景

设计系统（或任何知识推荐）需要**项目级持久化**：
- AI 助手的 context window 有限，不能每次都重新生成
- 不同页面/模块可能需要不同配置
- 团队成员需要共享统一的设计决策

---

## 模式定义

### Master + Overrides 模式

```
设计系统目录/
├── MASTER.md              # 项目级全局规则（Source of Truth）
└── pages/                 # 页面/模块级覆写
    ├── home.md            # 首页特殊规则
    ├── dashboard.md       # 仪表盘特殊规则
    ├── checkout.md        # 结账页特殊规则
    └── settings.md        # 设置页特殊规则
```

### 规则层级

```
优先级: pages/{page}.md  >  MASTER.md  >  默认值

加载逻辑:
  if pages/{current-page}.md 存在:
      rules = merge(MASTER.md, pages/{page}.md)  # page 覆盖 master
  else:
      rules = MASTER.md
```

## 页面覆写文件

### pages/checkout.md 示例

```markdown
# Page Override: Checkout

> Overrides MASTER.md for the checkout page only.

## Color Override
| Token | Hex | Reason |
|-------|-----|--------|
| --accent | #16A34A | Green for "Buy Now" to signal safety |
| --destructive | #DC2626 | Red for cancel/back actions |

## Style Override
- Reduce visual complexity
- No glassmorphism effects (distraction from purchase flow)
- Trust badges prominently placed

## Anti-Patterns (checkout-specific)
- ❌ Multiple CTAs competing for attention
- ❌ Surprise costs at last step
- ❌ Required account creation before purchase
```

### 覆写规则

```
MASTER.md 说:  accent = #EA580C (Orange)
checkout.md:   accent = #16A34A (Green)

合并结果: 在 checkout 页面用 Green，其他页面用 Orange
```

## 持久化 API

### 生成 MASTER

```python
def persist_master(design_system: dict, project_dir: str) -> str:
    """
    将设计系统持久化为 MASTER.md。

    Args:
        design_system: generate() 返回的完整设计系统字典
        project_dir: 项目根目录

    Returns:
        生成的文件路径
    """
    output_dir = os.path.join(project_dir, "design-system")
    os.makedirs(output_dir, exist_ok=True)

    filepath = os.path.join(output_dir, "MASTER.md")
    content = format_master_markdown(design_system)

    with open(filepath, "w", encoding="utf-8") as f:
        f.write(content)

    return filepath
```

### 生成页面覆写

```python
def persist_page(design_system: dict, page_name: str, project_dir: str) -> str:
    """
    为特定页面生成覆写文件。

    Args:
        design_system: 页面特定的设计系统（可能与 MASTER 不同）
        page_name: 页面名称（如 "checkout", "dashboard"）
        project_dir: 项目根目录

    Returns:
        生成的文件路径
    """
    output_dir = os.path.join(project_dir, "design-system", "pages")
    os.makedirs(output_dir, exist_ok=True)

    filepath = os.path.join(output_dir, f"{page_name}.md")
    content = format_page_override(design_system, page_name)

    with open(filepath, "w", encoding="utf-8") as f:
        f.write(content)

    return filepath
```

### CLI 命令

## AI 使用流程

### AI 读取持久化文件的决策树

```
AI 收到: "实现首页 hero section"
  │
  ├─ 检查 design-system/pages/home.md
  │   ├─ 存在 → 加载 MASTER.md + pages/home.md，合并
  │   └─ 不存在 → 只加载 MASTER.md
  │
  ├─ 提取: 配色、风格、字体、动效
  │
  └─ 按规则编码 → 输出代码
```

### Skill 文件中的指导

在 SKILL.md 中指导 AI 如何使用持久化文件：

```markdown
## Design System Persistence

When building pages:
1. Check `design-system/MASTER.md` for global design rules
2. Check `design-system/pages/{page-name}.md` for page-specific overrides
3. Page overrides take precedence over MASTER
4. If no design system exists, generate one first with --persist flag
```

## 适用场景

| 场景 | 使用 MASTER | 使用覆写 |
|------|------------|---------|
| 全站统一风格 | ✅ | ❌ |
| 页面有特殊需求 | ✅ 基础 | ✅ 差异部分 |
| A/B 测试变体 | ✅ 控制组 | ✅ 实验组 |
| 多品牌/多主题 | ✅ 默认品牌 | ✅ 其他品牌 |
