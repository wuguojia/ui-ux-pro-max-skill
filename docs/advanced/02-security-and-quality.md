# 02 — 安全性与质量保证

## 安全分析

### 攻击面

| 入口 | 风险等级 | 说明 |
|------|---------|------|
| CSV 数据文件 | 低 | 只读，不执行代码 |
| Python 搜索脚本 | 低 | 不接受网络输入 |
| CLI 安装器 | 中 | 下载 ZIP + 文件写入 |
| GitHub API 调用 | 低 | 只读，无认证 |
| 用户搜索输入 | 低 | 仅用于字符串匹配 |

### CSV 注入防护

CSV 数据被当作纯文本处理，不会被 `eval()` 或注入到 shell 命令中：

```python
# search() 函数只做字符串匹配:
for token in query_tokens:
    if token in document_tokens:
        score += idf[token] * tf_component
# 不存在注入风险
```

### CLI 安全措施

1. **ZIP 下载验证**: 只从 `api.github.com` 下载
2. **文件路径检查**: 不允许路径穿越 (`../`)
3. **无认证信息**: CLI 不要求用户提供 token
4. **Rate Limit**: 即使被限流也优雅降级

### 依赖安全

**Python 脚本**: 零外部依赖（只用标准库）
**CLI**: 仅 4 个依赖，全部为知名维护良好的包

| 依赖 | 周下载量 | 最后更新 | 已知漏洞 |
|------|---------|---------|---------|
| commander | 100M+ | 活跃 | 无 |
| chalk | 100M+ | 活跃 | 无 |
| ora | 10M+ | 活跃 | 无 |
| prompts | 20M+ | 稳定 | 无 |

## 质量保证

### 数据质量

**一致性检查**:
- `_sync_all.py` 确保 products/colors/reasoning 三表对齐
- 每个产品类型有且只有一条颜色记录
- 每个产品类型有且只有一条推理规则

**WCAG 色彩对比**:
- `_sync_all.py` 自动检查颜色对比度
- Accent 色会自动调整以满足 WCAG 3:1 标准
- 每条颜色记录的 Notes 会标注调整原因

### 搜索质量

**BM25 特性保证**:
- IDF 权重确保稀有词比常见词更重要
- 文档长度归一化避免长文档偏见
- k1=1.5 防止词频饱和
- 返回结果按相关性降序排列

### 代码质量

| 指标 | 状态 |
|------|------|
| TypeScript 严格模式 | CLI 使用 |
| Python 类型注解 | 部分使用 |
| 单元测试 | 未覆盖 |
| E2E 测试 | 未覆盖 |
| Linting | 未配置 |
