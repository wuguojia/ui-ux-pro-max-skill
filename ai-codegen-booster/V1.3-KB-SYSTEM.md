# AI 代码生成增强器 v1.3 - 知识库系统

## 🎉 新功能概述

v1.3 版本引入了完整的**知识库系统**，可以自动扫描项目代码，提取组件和样式信息，生成 AI 代码生成所需的知识库。

### 核心能力

- ✅ **自动扫描**：递归扫描项目目录，自动提取组件和样式
- ✅ **多框架支持**：React (.tsx, .jsx) 和 Vue (.vue)
- ✅ **样式提取**：CSS, Less, Sass/SCSS
- ✅ **项目级/全局级**：支持项目特定或全局共享知识库
- ✅ **CLI 工具**：完整的命令行工具管理知识库
- ✅ **合并模式**：可以合并多个源的知识库

## 🚀 快速开始

### 1. 初始化知识库

**项目级知识库**（推荐）：
```bash
# 在项目根目录
npm run kb:init

# 或直接使用 CLI
npm run kb init
```

**全局知识库**：
```bash
npm run kb init --global
```

这会创建配置文件：
- 项目级：`.kb-config.json` 在当前目录
- 全局级：`~/.ai-codegen-booster/kb-config.json`

### 2. 添加扫描源

```bash
# 添加 src/components 目录
npm run kb add-source \
  --name "components" \
  --path "./src/components" \
  --framework "React"

# 添加 Vue 组件目录
npm run kb add-source \
  --name "vue-components" \
  --path "./src/components" \
  --framework "Vue"

# 添加样式目录
npm run kb add-source \
  --name "styles" \
  --path "./src/styles" \
  --framework "Unknown"
```

### 3. 构建知识库

```bash
# 扫描所有源并生成 CSV
npm run kb:build

# 详细输出
npm run kb build --verbose

# 合并模式（不覆盖现有知识库）
npm run kb build --merge
```

构建完成后，会在 `./kb/` 目录生成：
- `components.csv` - 组件知识库
- `styles.csv` - 样式知识库
- `conventions.csv` - 编码规范（如果有）

## 📖 CLI 命令详解

### `kb init` - 初始化

初始化知识库配置。

```bash
npm run kb init              # 项目级
npm run kb init --global     # 全局级
npm run kb init --path ./my-project  # 指定路径
```

**效果**：
- 创建配置文件
- 创建输出目录
- 初始化空的源列表

---

### `kb add-source` - 添加源

添加要扫描的源目录。

```bash
npm run kb add-source \
  --name <源名称> \
  --path <目录路径> \
  --framework <框架类型> \
  [--global]
```

**参数**：
- `--name, -n`：源名称（必需）
- `--path, -p`：源路径（必需）
- `--framework, -f`：框架类型（React | Vue | Node | Unknown）
- `--global, -g`：添加到全局 KB

**示例**：
```bash
# React 组件库
npm run kb add-source -n "ui-components" -p "./packages/ui/src"  -f "React"

# Vue 组件目录
npm run kb add-source -n "vue-pages" -p "./src/pages" -f "Vue"

# 全局 antd 组件
npm run kb add-source -n "antd" -p "./node_modules/antd" -f "React" --global
```

---

### `kb build` - 构建知识库

扫描所有源并生成知识库。

```bash
npm run kb build [选项]
```

**选项**：
- `--verbose, -v`：详细输出
- `--merge, -m`：合并模式（不覆盖现有）
- `--global, -g`：构建全局 KB

**示例**：
```bash
# 标准构建
npm run kb build

# 详细输出模式
npm run kb build --verbose

# 合并到现有知识库
npm run kb build --merge

# 构建全局知识库
npm run kb build --global
```

**输出**：
```
🔍 Scanning sources...
📂 Scanning: components (./src/components)
   Components: 15
   Styles: 8
   Files scanned: 23

📝 Generating CSV files...

✅ Knowledge base built successfully in 1234ms
📊 Summary:
   Total components: 15
   Total styles: 8
   Total conventions: 0

📁 Output: ./kb
```

### `kb scan` - 快速扫描

快速扫描单个目录（不保存到配置）。

```bash
npm run kb scan --path <目录> [--output <输出目录>]
```

**参数**：
- `--path, -p`：要扫描的目录（必需）
- `--output, -o`：输出目录（默认：`./kb-scan`）
- `--verbose, -v`：详细输出

**示例**：
```bash
# 扫描 src 目录
npm run kb scan --path ./src

# 指定输出目录
npm run kb scan --path ./packages/ui --output ./ui-kb

# 详细模式
npm run kb scan --path ./src --verbose
```

## 🔧 配置文件格式

### 项目级配置 (`.kb-config.json`)

```json
{
  "mode": "project",
  "basePath": "/path/to/project",
  "outputDir": "./kb",
  "sources": [
    {
      "name": "components",
      "path": "./src/components",
      "framework": "React",
      "enabled": true
    },
    {
      "name": "styles",
      "path": "./src/styles",
      "framework": "Unknown",
      "enabled": true
    }
  ],
  "priority": "local-first",
  "autoSync": false
}
```

```json
{
  "mode": "global",

  "outputDir": "~/.ai-codegen-booster/global-kb",
  "sources": [
    {
      "name": "antd",
      "path": "/path/to/node_modules/antd",
      "framework": "React",
      "enabled": true
    }
  ],
  "priority": "local-first",
  "autoSync": false
}
```

## 🎯 使用场景

### 场景 1：单项目知识库

为当前项目生成专属知识库。

```bash
# 1. 初始化
cd /path/to/my-project
npm run kb init

# 2. 添加源
npm run kb add-source -n "src" -p "./src" -f "React"

# 3. 构建
npm run kb build

# 结果：在 ./kb/ 目录生成知识库
```

### 场景 2：全局组件库

为常用组件库（如 Ant Design）创建全局知识库。

```bash
# 1. 初始化全局 KB
npm run kb init --global

# 2. 添加 antd 源
npm run kb add-source \
  -n "antd" \
  -p "./node_modules/antd" \
  -f "React" \
  --global

# 3. 构建全局 KB
npm run kb build --global --verbose

# 结果：在 ~/.ai-codegen-booster/global-kb/ 生成全局知识库
```

### 场景 3：合并多个源

合并公司内部多个项目的组件。

```bash
# 添加多个源
npm run kb add-source -n "project-a" -p "../project-a/src"
npm run kb add-source -n "project-b" -p "../project-b/components"
npm run kb add-source -n "shared-ui" -p "./packages/ui"

# 构建合并
npm run kb build
```

### 场景 4：快速扫描第三方库

快速扫描第三方库但不保存配置。

```bash
# 扫描 element-plus
npm run kb scan \
  --path ./node_modules/element-plus \
  --output ./element-plus-kb \
  --verbose
```

## 🛠️ 编程接口

也可以直接在 Node.js 代码中使用：

```typescript
import { createKBManager, KnowledgeBaseScanner } from 'ai-codegen-booster/kb';

// 创建管理器
const manager = createKBManager({
  mode: 'project',
  basePath: process.cwd(),
  outputDir: './kb',
});

// 初始化
await manager.init('project');

// 添加源
await manager.addSource({
  name: 'components',
  path: './src/components',
  framework: 'React',
  enabled: true,
});

// 构建知识库
const kb = await manager.build({ verbose: true });

console.log(`提取了 ${kb.components.length} 个组件`);
console.log(`提取了 ${kb.styles.length} 个样式`);
```

### 直接使用扫描器

```typescript
import { KnowledgeBaseScanner } from 'ai-codegen-booster/kb';

const scanner = new KnowledgeBaseScanner({
  frameworks: ['React', 'Vue'],
  extensions: ['.tsx', '.vue', '.css'],
  verbose: true,
});

const result = await scanner.scan({
  name: 'my-source',
  path: './src',
  framework: 'React',
  enabled: true,
});

console.log(`扫描了 ${result.stats.filesScanned} 个文件`);
console.log(`找到 ${result.stats.componentsFound} 个组件`);
```

## 🔍 扫描规则

### 支持的文件类型

| 文件类型 | 扫描内容 |
|---------|---------|
| `.tsx`, `.jsx` | React 组件、Props、依赖 |
| `.vue` | Vue 组件、Props、Events、Slots、样式 |
| `.css` | CSS 变量、工具类 |
| `.less` | Less 变量、工具类 |
| `.scss`, `.sass` | Sass 变量、工具类 |

### 自动忽略的目录

- `node_modules/`
- `dist/`, `build/`
- `.git/`
- `coverage/`
- `__tests__/`, `__mocks__/`
- `*.test.*`, `*.spec.*`

### 提取的信息

**React 组件**：
- 组件名称
- Props（名称、类型、是否必需）
- 依赖的其他组件
- 导入路径

**Vue 组件**：
- 组件名称
- Props（名称、类型、是否必需、默认值）
- Events（事件名称）
- Slots（插槽名称）
- API 风格（Composition / Options）
- 组件内样式（CSS/Less/SCSS）

**样式**：
- 变量名和值
- 分类（颜色、间距、排版等）
- 预处理器类型
- 工具类和属性

## ⚙️ 高级选项

### 自定义忽略模式

编辑 `.kb-config.json`：

```json
{
  "sources": [
    {
      "name": "components",
      "path": "./src/components",
      "framework": "React",
      "include": ["**/*.tsx", "**/*.jsx"],
      "exclude": ["**/*.stories.*", "**/deprecated/**"],
      "enabled": true
    }
  ]
}
```

### 混合模式（Hybrid）

同时使用项目级和全局级知识库：

```json
{
  "mode": "hybrid",
  "priority": "local-first",
  "sources": [
    // 项目特定组件
    { "name": "local-components", "path": "./src" },
    // 全局共享组件
    { "name": "global-ui", "path": "~/.kb/ui-components" }
  ]
}
```

## 📝 最佳实践

### 1. 组织源

按功能或框架分组：

```bash
npm run kb add-source -n "react-ui" -p "./src/components/react"
npm run kb add-source -n "vue-pages" -p "./src/pages/vue"
npm run kb add-source -n "shared-styles" -p "./src/styles"
```

### 2. 定期更新

代码变更后重新构建知识库：

```bash
# 合并模式保留手动添加的条目
npm run kb build --merge
```

### 3. 版本控制

将 `.kb-config.json` 加入版本控制：

```bash
git add .kb-config.json
git commit -m "Add KB configuration"
```

但可以忽略生成的 KB：

```gitignore
/kb/
/kb-scan/
```

### 4. CI/CD 集成

在 CI 流程中自动生成知识库：

```yaml
# .github/workflows/kb.yml
name: Build Knowledge Base
on: [push]
jobs:
  build-kb:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install
      - run: npm run kb build
      - uses: actions/upload-artifact@v2
        with:
          name: knowledge-base
          path: kb/
```

## 🐛 常见问题

### Q: 扫描很慢怎么办？

A: 使用 `exclude` 排除大目录：

```json
{
  "exclude": ["**/node_modules/**", "**/dist/**", "**/*.min.*"]
}
```

### Q: 如何扫描 monorepo？

A: 为每个包添加单独的源：

```bash
npm run kb add-source -n "pkg-a" -p "./packages/package-a/src"
npm run kb add-source -n "pkg-b" -p "./packages/package-b/src"
```

### Q: 可以扫描 node_modules 吗？

A: 可以，但要明确指定路径：

```bash
npm run kb add-source \
  -n "antd" \
  -p "./node_modules/antd/es" \
  -f "React"
```

### Q: 如何自定义提取逻辑？

A: 直接使用编程接口并扩展扫描器。
