# AI Codegen Booster - 快速参考

## 常用命令

| 你说... | AI会做什么 |
|--------|-----------|
| **分析项目** | 扫描整个项目，提取组件、样式、编码规范 |
| **分析 src/components** | 只分析组件目录 |
| **分析 src/styles** | 只分析样式文件 |
| **提取React组件** | 提取React组件的Props、Hooks、Context |
| **提取Vue组件** | 提取Vue组件的Composables、响应式变量 |
| **检测设计模式** | 识别单例、工厂、观察者等模式 |
| **检测架构模式** | 识别MVC、MVVM、Clean Architecture |
| **优化知识库** | 去重、评分、修复不一致 |
| **质量评分** | 评估知识库质量（0-100分） |
| **创建XX组件** | 使用知识库生成符合规范的代码 |
| **验证这段代码** | 检查代码是否符合项目规范 |

## 知识库文件位置

```
your-project/
└── kb/
    ├── components.csv      # 组件库
    ├── styles.csv         # 样式库
    ├── conventions.csv    # 规范库
    └── quality-report.md  # 质量报告
```

## 提取的信息

### 组件信息
- ✅ 组件名称、框架、导入路径
- ✅ Props（类型、描述、默认值）
- ✅ 事件、插槽、泛型
- ✅ React Hooks（useState, useEffect, 自定义Hook）
- ✅ Vue Composables（ref, reactive, computed）
- ✅ Context/Provide-Inject 模式
- ✅ 使用示例代码

### 样式信息
- ✅ CSS变量、工具类
- ✅ 布局模式（Flexbox, Grid）
- ✅ 语义化标签
- ✅ 可访问性模式（ARIA）

### 编码规范
- ✅ 命名约定（PascalCase, camelCase等）
- ✅ 设计模式（单例、工厂、观察者等）
- ✅ 架构模式（MVC、MVVM等）
- ✅ 最佳实践
- ✅ 反模式检测（长函数、深层嵌套、魔法数字）

## 典型工作流

### 1️⃣ 首次使用
```
"分析整个项目，生成知识库"
```

### 2️⃣ 生成代码
```
"创建一个用户资料卡片组件"
```
→ AI会使用知识库中的组件和样式

### 3️⃣ 增量更新
```
"扫描新增的组件，更新知识库"
```

### 4️⃣ 质量检查
```
"评估知识库质量"
"优化知识库，去除重复"
```

## 质量评分指标

- **整体质量** (0-100分)
  - 完整性：文档是否完整
  - 一致性：命名、路径是否统一
  - 可复用性：是否易于理解和使用
  - 文档性：描述和示例是否充分
